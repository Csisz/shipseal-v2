# ShipSeal Agent Instructions

ShipSeal is an AI Project Delivery Pack Generator. Agents working in this repository should keep changes small, local-first, deterministic, and easy to validate.

## Required Commands

Run these before handing work back:

```bash
npm run test
npm run build
```

Use `npm run lint` when touching shared UI, build config, or TypeScript surfaces. Existing lint warnings should be reviewed, but do not use broad automated fixes without human approval.

## Product Boundaries

- Do not add auth, payments, private GitHub import, GitHub App integration, AI2AI integration, databases, persistent storage, or server-side PDF services without explicit human approval.
- Do not call external AI APIs from this MVP.
- Keep ZIP upload and public GitHub import local-first/browser-safe.
- Uploaded or imported code must never be executed.

## Delivery Pack Export

The Delivery Pack export is manifest-based. Required outputs live under `src/lib/deliveryPack/manifest.ts` and are generated by the production generator path. Do not change folder structure, ZIP paths, `score.json`, or generated output contracts without tests.

When changing export behavior, verify:

- Delivery Pack ZIP still includes every manifest path.
- `06-client-handoff/CLIENT_HANDOFF_REPORT.md` remains.
- `06-client-handoff/CLIENT_HANDOFF_REPORT.html` remains.
- `score.json` remains valid JSON.
- ZIP filename starts with `shipseal-delivery-pack-`.

## GitHub Import

Public GitHub import is proxy-first in hosted/Vercel mode:

```text
/api/github-archive?owner=Csisz&repo=shipseal&ref=HEAD
```

Direct `codeload.github.com` import may be used only as fallback. If browser restrictions block direct download, the UI must recommend downloading the repository ZIP from GitHub and uploading it manually.

## Print-Ready And PDF Report

The client report source of truth is the standalone HTML report generated from ShipSeal report data and intake. The PDF export should be generated from that standalone client report, not from dashboard printing.

If PDF generation fails, preserve the fallback:

```text
PDF generation failed. Open the HTML report and use Print / Save as PDF.
```

## Human Review Required

Require human review before changing:

- `src/lib/scanner.ts`
- `src/lib/scannerLimits.ts`
- `src/lib/exports.ts`
- `src/lib/deliveryPack/**`
- `src/lib/report/**`
- `api/github-archive.ts`
- `package.json`
- `package-lock.json`
- `vercel.json`
- `.github/workflows/**`

## Legal And Security Language

ShipSeal must not claim legal compliance, AI Act certification, or production security audit status. Use clear disclaimers:

- "This is not legal advice."
- "ShipSeal does not provide legal advice."
- "Security output is not a full production security audit."

## Working Style

- Prefer small, test-backed changes.
- Do not rewrite business logic or scanner logic unless requested.
- Keep UI copy client-friendly and understandable for AI freelancers and small agencies.
- Preserve ZIP upload as the reliable fallback path.
