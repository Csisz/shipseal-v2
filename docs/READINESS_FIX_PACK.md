# ShipSeal Readiness Fix Pack

The Readiness Fix Pack is a repository-facing ZIP generated by ShipSeal.

It is different from the ShipSeal Delivery Pack.

- Delivery Pack: client handoff package with reports, AI Act readiness, testing pack and agent instructions.
- Readiness Fix Pack: repository files you can add back to your project to improve future scans and make the repo more agent-ready.

The current MVP can either export the ZIP for manual application or create a reviewed GitHub pull request when the user explicitly provides a token and confirms the operation. ShipSeal never pushes directly to `main`.

## Files Included

Minimum files:

- `AGENTS.md`
- `CLAUDE.md`
- `CONTRIBUTING.md`
- `SECURITY.md`
- `docs/CRITICAL_FILES_POLICY.md`
- `docs/RELEASE_CHECKLIST.md`
- `docs/OWNERSHIP.md`
- `.github/workflows/ci.yml`

Additional MVP files may include:

- `docs/AGENT_CHANGE_POLICY.md`
- `docs/HANDOFF_CHECKLIST.md`
- `docs/AI_ACT_READINESS_NOTES.md`

## What The Files Contain

The generated files include:

- project name,
- detected stack,
- detected build/test/lint commands where available,
- AI coding agent workflow rules,
- files that should not be changed without human review,
- testing and release validation steps,
- security and secret handling rules,
- GitHub import and Delivery Pack validation reminders,
- cautious AI Act readiness notes where relevant.

AI Act readiness notes are preliminary technical/product-side notes only. They are not legal advice.

## How To Apply The Fix Pack

1. Run a ShipSeal scan.
2. Open the Suggested Readiness Fix Pack section.
3. Click `Download Readiness Fix Pack`.
4. Create a branch in your repository.
5. Unzip `shipseal-readiness-fix-pack-[repo].zip` into the repository root.
6. Review every generated file.
7. Commit and open a pull request.

```bash
git checkout -b shipseal/readiness-pack

# unzip shipseal-readiness-fix-pack-[repo].zip into the repository root

git add AGENTS.md CLAUDE.md CONTRIBUTING.md SECURITY.md docs/ .github/workflows/ci.yml
git commit -m "Add ShipSeal readiness fix pack"
git push origin shipseal/readiness-pack
```

Then open a Pull Request on GitHub.

## Why A Separate Branch

A separate branch keeps generated files reviewable and reversible. It lets CI run and gives humans a clean place to approve or reject each repository change.

ShipSeal should not push directly to `main`. The Create Readiness PR MVP creates a separate branch and opens a pull request for human review.

## Create Readiness PR

The Readiness Fix Pack defines the file manifest, generated content, metadata and manual workflow that `Create Readiness PR` reuses.

The MVP flow:

1. Shows the planned files and workflow warning.
2. Requires a GitHub token only for the submit request.
3. Creates `shipseal/readiness-pack` or a timestamped fallback branch.
4. Uploads the reviewed generated files.
5. Opens a pull request.
6. Leaves merge decisions to humans.
