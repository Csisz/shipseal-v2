# ShipSeal Book
## The Source of Truth

**Version:** 1.0 Draft

## Purpose

This document is the single source of truth for the ShipSeal product.

Every roadmap, backlog, landing page, report, feature, prompt and design decision should align with this document.

## Vision

**ShipSeal is an AI Workspace Optimizer.**

ShipSeal transforms software repositories into AI-optimized workspaces that coding agents can understand faster, navigate more efficiently and improve with dramatically less wasted context.

ShipSeal does not compete with Claude Code, Codex or Cursor.

ShipSeal amplifies them.

## Mission

Help every AI coding agent become dramatically more productive by improving repository understanding rather than model intelligence.

## Core Insight

AI models improve every year.

Repositories become larger, noisier and more fragmented.

The bottleneck is no longer intelligence.

The bottleneck is context.

ShipSeal exists to solve the context problem.

## Product Definition

ShipSeal is not:
- an IDE
- an LLM
- a chatbot
- a CI platform
- a project management tool

ShipSeal is:
- an AI Workspace Optimizer
- a Repository Intelligence Layer
- a Project Memory Generator
- an Agent Productivity Platform

## Product Pillars

1. AI Workspace Foundation
2. Project Memory
3. Context Compression
4. Agent Routing
5. Agent Productivity Analytics
6. Delivery Automation
7. Enterprise Add-ons

## Product Principles

- Repository before Model
- Context before Intelligence
- Evidence before Assumptions
- Compression before Expansion
- Human-readable first
- AI-native by design
- Deterministic whenever possible
- Every generated file must have a purpose

## Primary Customer

Primary: Claude Code users.

Secondary:
- AI-first startups
- AI agencies
- Engineering teams

## Core Metrics

Primary:
- AI Workspace Quality
- Agent Productivity
- Context Efficiency
- Project Memory Coverage

Supporting:
- Repository Health
- Delivery Readiness
- MCP Readiness

## Long-term Vision

Every serious repository should eventually contain a ShipSeal Workspace Layer, just as repositories today contain README files and CI pipelines.
