# MOTION_LANGUAGE.md

> **Turn software into knowledge.**

## Purpose

Motion in ShipSeal is not decoration. Motion exists to make repository understanding visible.

## Core Principle

Every animation must represent a real repository event or a real product state.

Animate when it helps the user understand that:

- something was discovered,
- something was connected,
- something became clearer,
- something is ready,
- something needs attention.

## Emotional Target

ShipSeal motion should feel calm, premium, intelligent, deliberate, trustworthy and alive.

It should never feel noisy, random, childish, overwhelming or decorative.

## Motion Rules

### 1. Reveal instead of dump

Information should not appear all at once. Repository understanding should build gradually.

### 2. Understanding before metrics

Scores and numbers should appear after the user sees what ShipSeal understood.

Preferred order:

1. discovery
2. connection
3. meaning
4. confidence
5. metrics

### 3. Motion explains relationships

When repository signals are related, animate a line, glow, orbit, pulse or path between them.

Examples:

- README connects to Documentation.
- AGENTS.md connects to Project Memory.
- Tests connect to Verification.
- Build commands connect to Developer Workflow.
- Generated folders connect to Context Compression.

### 4. No decorative motion

Avoid animations that do not communicate meaning.

### 5. Short, intentional timing

Suggested timings:

- micro interaction: 120–180 ms
- card reveal: 180–260 ms
- node reveal: 220–360 ms
- graph connection: 300–500 ms
- major transition: 600–900 ms
- dramatic pause before reveal: 600–1000 ms

## Signature Motion Patterns

### Intelligence Reveal™

Used after scan completion.

Flow:

1. dark calm canvas
2. central intelligence pulse
3. repository signals appear one by one
4. connections form
5. short pause
6. “Repository understood.”
7. AI Workspace fades in

### Repository DNA™ Build

Used when showing the AI Workspace profile.

Flow:

1. empty radar/profile frame
2. each axis grows independently
3. evidence badges appear
4. current shape fills
5. potential shape appears as subtle secondary layer
6. selected dimension opens

### Mental Model™ Connection

Used when showing semantic repository understanding.

Flow:

1. central repository node appears
2. major knowledge nodes appear
3. evidence-backed connections draw in
4. weak or heuristic links appear more subtly
5. selected nodes pulse on hover

### Living Repository™ Pulse

Used for the future repository universe.

Flow:

1. repository nodes slowly orbit
2. important nodes glow more strongly
3. missing signals remain dim
4. warnings pulse gently
5. user can zoom or inspect

## Evidence States

### Evidence-backed

- brighter glow
- solid connection
- confident reveal
- “Evidence” badge

### Heuristic

- softer glow
- dashed or thinner connection
- slower reveal
- “Heuristic” badge

### Missing

- dim node
- no glow
- optional warning accent
- recommendation appears

## Screen-level Motion

### Landing

Minimal. Clarity over spectacle.

### Scan

Most important motion surface. The scan should feel like understanding, not loading.

### Post-scan

The first post-scan moment should be cinematic. The dashboard should not appear before the user sees what ShipSeal learned.

### Delivery Outputs

Restrained. Delivery is practical and secondary.

## Anti-patterns

Avoid:

- spinning loaders as the primary scan experience
- endless pulsing without meaning
- too many glowing elements
- animated KPI walls
- instant dashboard dump after scan
- flashy sci-fi effects without evidence
- animations that hide important information

## Accessibility

- respect reduced-motion preferences where possible
- avoid rapid flashing
- keep text readable
- do not require animation to understand critical content
- keep actions available after transitions complete

## Product Rule

A motion pattern is accepted only if it increases understanding or delight.

If it does neither, remove it.

## Definition of Done

A ShipSeal motion experience is successful when:

- the user understands what changed
- the motion communicates repository intelligence
- the screen feels calmer after the animation, not busier
- the product feels more alive
- the experience supports the motto: **Turn software into knowledge**
