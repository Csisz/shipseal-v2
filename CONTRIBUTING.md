## Contribution Workflow

1. Create a topic branch for every change.
2. Keep changes small and focused.
3. Run the available verification commands before opening a pull request.
4. Include a short summary, risk notes and test results in the PR description.
5. Ask for human review before merging AI-assisted changes.

## Verification Commands

- `bun install`
- `bun run lint`
- `bun run test`
- `bun run build`
- Download the PDF client report and review the Delivery Pack export before client-facing release.

## AI-Assisted Contributions

- AI-generated changes must be reviewed by a human maintainer.
- Do not let agents modify secrets, production config, payment/auth flows or legal/compliance language without explicit review.
- Include ShipSeal score and Delivery Pack observations when the change affects handoff readiness.
